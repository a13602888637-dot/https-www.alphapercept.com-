## 2. 自选股功能修复

### 问题描述
自选股功能无法使用，需要支持股票代码和名称搜索添加。

### 设计方案
**目标**：修复自选股添加功能，使用新浪/腾讯数据源

**数据源**：
- 腾讯财经：股票搜索和基本信息
- 新浪财经：股票实时数据

**组件结构**：
1. **搜索组件** (`SearchStockInput`)
   - 支持股票代码和名称输入
   - 实时搜索建议
   - 防抖处理（300ms）

2. **搜索结果列表** (`SearchResultsList`)
   - 显示匹配的股票
   - 股票代码、名称、市场信息
   - 添加按钮

3. **自选股列表** (`Watchlist`)
   - 显示已添加的股票
   - 实时价格更新
   - 删除功能

**API集成**：
```typescript
// 使用现有技能文件
import { searchStocksFromTencent } from '@/skills/data_crawler'
import { getStockRealTimeData } from '@/skills/data_crawler'
```

**数据流**：
1. 用户输入搜索词 → 调用腾讯财经搜索API
2. 显示搜索结果 → 用户选择股票
3. 添加到自选股 → 保存到本地状态
4. 实时更新 → 使用新浪财经数据

**存储方案**：
- 短期：localStorage
- 长期：集成Prisma数据库（后续）

## 3. Vercel部署

### 问题描述
需要将项目部署到公网，使用Vercel方案。

### 设计方案
**目标**：配置Vercel部署，支持生产环境运行

**部署步骤**：
1. **环境准备**
   ```bash
   npm i -g vercel
   vercel login
   ```

2. **项目链接**
   ```bash
   vercel link
   ```

3. **环境变量配置**
   - 复制`.env.local`到Vercel环境变量
   - 关键变量：
     - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
     - `CLERK_SECRET_KEY`
     - `NEXT_PUBLIC_APP_URL`

4. **部署命令**
   ```bash
   vercel --prod
   ```

**Vercel配置**：
- 构建命令：`npm run next:build`
- 输出目录：`.next`
- Node版本：18.x
- 环境变量：从`.env.local`导入

**持续部署**：
- 连接GitHub仓库
- 自动部署main分支
- 预览部署PR分支
