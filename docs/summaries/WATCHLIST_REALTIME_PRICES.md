# Watchlist Real-time Prices Integration

## 已完成的功能

### 1. 新的API端点
- **`/api/stock-prices`**: 获取实时股票价格数据
  - 支持批量查询多个股票代码
  - 返回完整的市场数据（价格、涨跌幅、成交量等）
  - 内置错误处理和回退机制

### 2. 修改的组件

#### WatchlistManager组件 (`/components/watchlist/WatchlistManager.tsx`)
- **替换了模拟价格生成器**：原来的`fetchStockPrices`函数使用随机价格生成，现在调用真实API
- **添加了完整的价格数据结构**：支持存储价格、涨跌幅、成交量等完整信息
- **实现了定时刷新**：每30秒自动更新价格数据
- **添加了错误处理**：API失败时使用买入价格作为回退

#### WatchlistCard组件 (`/components/watchlist/WatchlistCard.tsx`)
- **更新了价格显示**：显示当前价格和今日涨跌幅
- **改进了盈亏计算**：基于真实市场价格计算盈亏
- **添加了更新时间显示**：显示价格最后更新时间

### 3. 数据流
```
用户添加股票到自选股 → WatchlistManager获取股票列表 → 调用stock-prices API →
从新浪/腾讯/Tushare获取实时数据 → 更新UI显示
```

## 技术实现

### API集成
- 使用现有的`data_crawler.ts`模块中的`fetchMultipleStocks`函数
- 支持新浪财经、腾讯财经、Tushare多个数据源
- 自动故障转移：主数据源失败时使用备用数据源

### 错误处理
1. **API调用失败**：使用买入价格作为回退值
2. **单个股票数据缺失**：不影响其他股票显示
3. **网络超时**：设置合理的超时时间（10秒）

### 性能优化
1. **批量查询**：一次API调用获取所有股票价格
2. **缓存策略**：30秒刷新间隔避免频繁请求
3. **按需加载**：只在有自选股时发起请求

## 支持的股票代码格式

- **A股代码**：6位数字，如`000001`、`600000`
- **自动识别交易所**：代码以6开头为上证，其他为深证
- **批量查询**：支持逗号分隔的多个代码

## 数据字段说明

返回的价格数据包含以下字段：
- `price`: 当前价格
- `change`: 涨跌额
- `changePercent`: 涨跌幅百分比
- `high`: 今日最高价
- `low`: 今日最低价
- `volume`: 成交量（股）
- `turnover`: 成交额（元）
- `lastUpdate`: 最后更新时间
- `name`: 股票名称

## 故障排除

### 常见问题

1. **价格显示为0或买入价格**
   - 检查网络连接
   - 确认股票代码正确
   - 查看浏览器控制台错误信息

2. **API返回错误**
   - 检查开发服务器是否运行
   - 查看服务器日志
   - 测试API端点是否可访问

3. **价格不更新**
   - 检查定时器是否正常工作
   - 确认watchlist不为空
   - 查看网络请求是否成功

### 日志查看

