## 测试步骤

### 步骤1：启动开发服务器
```bash
npm run dev
```

### 步骤2：访问应用并登录
1. 打开浏览器访问 `http://localhost:3000`
2. 点击登录按钮，使用Clerk认证登录

### 步骤3：访问自选股功能
1. 登录后进入仪表板
2. 点击左侧菜单或顶部标签页中的"自选股管理"

### 步骤4：测试添加功能
1. 点击"添加股票"按钮
2. 填写股票信息：
   - 股票代码：`000001`
   - 股票名称：`平安银行`
   - 买入成本：`12.50`（可选）
   - 止损价：`10.20`（可选）
   - 目标价：`15.00`（可选）
   - 备注：`测试股票`（可选）
3. 点击"添加"按钮
4. 验证：应该看到成功提示"股票已添加到自选股"

### 步骤5：测试搜索功能
1. 在搜索框中输入"平安"
2. 验证：应该显示包含"平安"的股票
3. 在搜索框中输入"茅台"
4. 验证：如果没有茅台股票，应该显示"未找到匹配的股票"
5. 点击清除搜索按钮（×）
6. 验证：搜索框应该被清空，显示所有股票

### 步骤6：测试更新功能
1. 找到刚刚添加的股票
2. 点击编辑按钮（铅笔图标）
3. 修改价格信息
4. 点击保存按钮
5. 验证：应该看到成功提示"自选股已更新"

### 步骤7：测试删除功能
1. 点击删除按钮（垃圾桶图标）
2. 确认删除提示
3. 验证：应该看到成功提示"自选股已删除"

### 步骤8：测试错误场景
1. **未登录访问**：登出后尝试访问自选股页面
   - 预期：显示登录提示或401错误

2. **重复添加**：尝试添加已存在的股票
   - 预期：显示"该股票已在自选股中"

3. **空数据添加**：尝试不填写股票代码和名称
   - 预期：显示"请输入股票代码和名称"

## 常见问题排查

### 问题1：无法添加股票
**可能原因**：
1. 用户未登录
2. 数据库连接问题
3. Clerk用户同步失败

**解决方案**：
1. 检查用户是否已登录
2. 检查数据库连接字符串
3. 检查Clerk webhook配置

### 问题2：搜索不工作
**可能原因**：
1. 搜索逻辑错误
2. 数据未正确加载

**解决方案**：
1. 检查搜索过滤逻辑
2. 检查数据加载状态

### 问题3：页面加载缓慢
**可能原因**：
1. 数据库查询慢
2. 网络延迟

**解决方案**：
1. 添加加载状态指示器
2. 优化数据库查询
3. 添加缓存机制
